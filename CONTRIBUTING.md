# Contributing

Thanks for contributing to the Duilio FPV ecosystem.

This repository is the **public hub** of the project.  
Its purpose is to provide a clear entry point to the ecosystem, explain how the parts connect, and direct contributors to the correct repository.

---

## Purpose Of This Repository

Use this repository for changes related to:

- ecosystem overview and project presentation
- hub-level documentation
- repository structure and navigation
- public-facing links and references
- contributor guidance for the overall ecosystem

Typical examples:

- improving the main README
- clarifying how onboard, offboard, and hardware repositories relate to each other
- fixing broken links
- improving onboarding for new contributors

---

## Where To Contribute

Please open code-specific issues and pull requests in the relevant repository:

- `duilio-fpv-onboard` for onboard software
- `duilio-fpv-offboard` for offboard software
- `duilio_f4` for hardware, firmware, and board-related work

Use this hub repository only for changes that affect the ecosystem at a higher level.

---

## What Does Not Belong Here

Please do **not** use this repository for:

- onboard code changes
- offboard code changes
- hardware or firmware implementation details
- detailed technical documentation that belongs in a subproject repository
- bug reports that are specific to one component

If a change mainly affects one subproject, it should be opened in that subproject repository.

---

## Before Opening An Issue Or Pull Request

Before opening a pull request here:

- confirm that the change belongs to the hub repository
- keep the change focused and limited in scope
- keep documentation concise, accurate, and easy to navigate
- avoid duplicating detailed content that should live in subproject repositories
- preserve this repository as a clean and simple ecosystem entry point

---

## Pull Request Guidelines

Good pull requests for this repository are usually small, clear, and focused.

Examples of suitable pull requests:

- improve the ecosystem overview
- clarify repository roles
- improve contributor instructions
- update external links
- fix wording, formatting, or navigation in hub documentation

When possible:

- use clear commit messages
- avoid unrelated cleanup in the same pull request
- keep naming and terminology consistent across repositories

---

## Cross-Repository Changes

If your contribution affects more than one repository:

- open the implementation changes in the relevant subproject repositories
- use this repository only for the ecosystem-level documentation or navigation update
- reference related issues or pull requests when useful

This helps keep responsibilities clear across the project.

---

## Questions

If you are unsure where a change belongs, open an issue in this repository first with a short description of the proposed change.

If the change is better suited to another repository, it can then be redirected cleanly.
